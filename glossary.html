Catalog
The Catalog is the name of the internal database of Keypirinha where all the items are indexed to allow a quick search.
CatalogItem
The class name of an item.
Hive
Hives
A Hive is a directory dedicated to the storage of packages. It also defines the way Packages are stored (i.e.: loose files or packed in a single archive).
InstanceID

The InstanceID number is an unsigned integer internally used by the application and assigned at runtime to every loaded package. It is used to ensure Plugins are not messing up the catalog.

Read Overview for more information.

Item
Items
An item is a simple data structure created by the plugins and that holds some properties like a display name (label), a category (File, URL, Command Line, ...), a reference to the parent plugin, etc...
LaunchBox

The L(a)unchBox is Keypirinha‘s main dialog window, from which you can make a search and launch the selected item. It can be opened by pressing the Ctrl+Win+K combination (by default).

Package
Packages

A Package is a container of one or several plugins designed for the same purpose. It contains the Plugins code as well as their dependencies if needed and optionally includes their configuration files, resources and documentation.

Its content can be stored in a directory or packed in an archive (e.g.: Zip) to ease redistribution.

Read Packages for more information.

Plugin
Plugins
In the context of Keypirinha, a Plugin is a software component written in Python3 that extends the base features of the application. It is always encapsulated in a package.
